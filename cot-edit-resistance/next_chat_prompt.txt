**Project summary:**

We started with Neel Nanda's CoT edit resistance problem. The core question: when a reasoning model's chain-of-thought is corrupted mid-generation by a fake final answer, does it repair?

**Experiment 1 (complete):** DeepSeek-R1-Distill-Qwen-1.5B, MATH-500 level 3, 55 correct traces.
Injected \boxed{0} at 30% of each reasoning trace, regenerated, measured repair.
- 49% repaired, 94.5% explicit backtracking ("wait", "actually")
- Key finding: "accepted_fake" was a mislabeled metric — 26/27 non-repair cases NEVER output \boxed{0}, they just looped until the 4096-token limit. The model is never fooled. It just can't exit the loop.
- Real split: repaired (committed correctly) vs. timed_out (looped to limit).

**Experiment 2 (complete, 196 runs):** DeepSeek-R1-Distill-Qwen-7B, MATH-500 level 3.
Swept injection depth (0.10, 0.30, 0.70) × injection type (absurd=\boxed{0}, correct=\boxed{ground_truth}).
Results at n=48 per condition (depths 0.10 and 0.70):
- Absurd injection: 98% detection (explicit backtrack), 31% correction at depth 0.10, 51% at depth 0.70
- Correct injection: ~87% repair, ~12% timeout even with right answer injected
- Detection is depth-independent. Correction scales with how much reasoning was completed.
- 0% accepted_fake (with fixed metric) — the model never actually commits to the wrong answer.
Data: data/experiment2_results.json

**Experiment 3 (in progress, pilot 3 problems):** Commitment probe + logit lens.
For each correct trace at depths 0.1-0.9:
- Truncate reasoning at depth X
- Append "\n\nFinal answer: $\boxed{" (forces model to complete the box)
- Forward pass with output_hidden_states=True → logit lens at last reasoning token position
- model.generate (30 tokens) → parse probe answer
- Logit lens: for each of 28 layers, record P(correct_answer_token) at the last reasoning token position

Pilot finding (3 problems):
- idx=1 (divisors of 196, gt=9): probe correct at ALL depths (0.1-0.9). Layer 26 jumps from ~0% to 99.85% for "9" — sharp phase transition in last 2 layers. Answer is computed very late in the network.
- idx=2 (perfect cube, gt=27): correct from depth 0.2 onwards, wrong at 0.1.
- idx=0 (\frac{14}{3}): broken ground truth in stored data (truncated to \frac{14}).

Key fix applied this session: changed logit lens position from last token of input (last token of commitment prompt) to last token of the reasoning prefix. The `{` token of `$\boxed{` was giving inconsistent results because it's sensitive to what immediately precedes it.
